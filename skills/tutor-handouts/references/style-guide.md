# Style Guide вЂ” Typography, Palette, Layout

Applies to every `.tex` artifact emitted under `Coursepack/Handouts/` and
`Coursepack/Syllabus/`.

## INV-2 (No Commercial Fonts)

The skill MUST NOT require any commercial font. The mandatory body font is
Latin Modern (`lmodern`), loaded by `commonheader.sty`.

If a future palette adds Stanford-cardinal or MIT-cardinal styling, it MAY
optionally swap in `mathpazo` (Palatino-style, free) but MUST keep `lmodern`
as the fallback for users on minimal LaTeX installs.

## INV-3 (No minted)

The skill MUST use `listings` for code blocks. `minted` is forbidden because
it requires Pygments + `--shell-escape`, both of which are deployment
friction.

## Palette Catalog

`commonheader.sty` defines five palettes. The skill rewrites
`\selectedpalette` at TH2 to apply the user's choice.

| Palette | Accent (RGB) | Use case |
|---|---|---|
| `generic-blue` | 28, 69, 135 | Default; neutral academic, no school affiliation |
| `cmu-red` | 198, 12, 48 | CMU 15-418-style |
| `stanford-cardinal` | 140, 21, 21 | Stanford CS149-style |
| `mit-cardinal` | 163, 31, 52 | MIT 6.172-style |
| `berkeley-blue` | 0, 50, 98 | Berkeley CS267-style |

Adding a new palette: define `<name>-accent`, `<name>-dark`, `<name>-light`
in `commonheader.sty`'s color block, then make it selectable via
`\selectedpalette{<name>}`.

## Layout Defaults

- Document class: `article` 11pt
- Page size: `letterpaper`
- Margins: 1 inch all sides (overridden to 0.4 in by `cheatsheet.cls`)
- Single column (except cheatsheet which uses `multicol` 2-col)
- Header: `fancyhdr` with accent-colored rule
- Footer: page number `N of M` via `lastpage`

## Section Style

Sections render with a colored rule underneath the title:
```
1 Section Title
в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
```

The rule is colored `\accentcolor`. Provided by `titlesec`.

## Code Listing Style

- Font: `\ttfamily\small`
- Frame: single
- Numbers: left, gray
- Comments: italic gray
- Strings: `\accentcolor`
- Keywords: `\bfseries\accentdark`
- Caption position: bottom (`captionpos=b`)
- Custom languages: `CUDA` and `Triton` defined in `commonheader.sty`

Examples:
```latex
\begin{lstlisting}[language=CUDA, caption={Vector add вЂ” CUDA C++ Guide В§2.2}]
__global__ void vector_add(...) { ... }
\end{lstlisting}

\begin{lstlisting}[language=Triton, caption={Vector add вЂ” Triton tutorial 01}]
@triton.jit
def vector_add_kernel(...): ...
\end{lstlisting}
```

## Inline Code

Use `\code{...}` macro (defined in `commonheader.sty`, wraps `\lstinline`).
This is the ONLY way inline API names should appear in prose.

## Table Style

Always `booktabs` (no vertical rules, three horizontal rules). Use
`tabularx` for tables that need a stretching column:

```latex
\begin{tabularx}{\linewidth}{lXr}
  \toprule
  Item & Description & Pts \\
  \midrule
  ... & ... & 10 \\
  \bottomrule
\end{tabularx}
```

## Math Style

- Display math via `\[ ... \]` or `align` (from `amsmath`)
- Inline math via `\(...\)`
- AMS theorem environments via `amsthm`
- `mathtools` for `\DeclarePairedDelimiter` etc.

## Hyperref Style

`hyperref` is loaded last with `colorlinks=true`. Link colors:
- Internal `\ref` / `\cite`: `\accentdark`
- External URLs: `\accentcolor`
- PDF outline visible

## Anti-AI-Slop Typography

- No emoji in prose or section titles
- No bold for emphasis in body text (italics only; bold reserved for run-in headings and table headers)
- No ALL-CAPS in body text (use `\textsc{}` for small caps if absolutely needed)
- No more than 2 levels of section nesting (`\section`, `\subsection`); deeper levels are forbidden in the `handout.cls` setup

## Generated-At Header

Every `.tex` file MUST start with:
```latex
% Generated by tutor-handouts at <ISO8601> вЂ” manual edits will be lost on re-run.
\documentclass[lecture]{handout}
```

Same convention in `.py` (using `#`), `Makefile` (using `#`), `.md` (using
HTML comment).
